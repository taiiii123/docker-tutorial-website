# レスポンシブデザイン完全対応仕様

## 背景・目的

スマートフォン・タブレットでの閲覧体験を最適化し、すべてのデバイスで快適に学習できるようにする。

## 対象デバイス

| デバイス | 画面幅 | Tailwindプレフィックス |
|---------|--------|----------------------|
| スマホ（縦） | 〜639px | なし（デフォルト） |
| スマホ（横）・小型タブレット | 640px〜767px | `sm:` |
| タブレット | 768px〜1023px | `md:` |
| タブレット（横）・小型PC | 1024px〜1279px | `lg:` |
| デスクトップ | 1280px〜 | `xl:` |

## 機能要件

### 1. 目次（TableOfContents）

**現状**: xl (1280px) 以上でのみ表示

**改善**:
- タブレット（md以上）: 折りたたみ可能なフローティングボタンで表示
- スマホ: ページ上部に折りたたみ式で表示
- デスクトップ（xl以上）: 従来通りサイドバー表示

### 2. コードブロック

**改善**:
- スマホ: フォントサイズを小さく（text-xs）
- 横スクロール時のインジケーター表示
- コピーボタンのタッチ領域拡大（min-height: 44px）

### 3. ヘッダー検索

**現状**: md以上でのみ表示

**改善**:
- スマホ: 検索アイコンタップでモーダル表示

### 4. セクションページ

**改善**:
- ブックマーク・完了ボタンのタッチ領域拡大
- パンくずリストの省略表示改善
- ナビゲーションボタンの固定表示（スマホ）

### 5. タッチ操作最適化

- タップ可能要素の最小サイズ: 44x44px
- ホバー状態の代替（:active疑似クラス）
- スワイプジェスチャーは対象外（複雑化回避）

## 非機能要件

- パフォーマンス: LCP 2.5秒以内
- アクセシビリティ: WCAG 2.1 AA準拠

## 受け入れ条件

- [ ] スマホ（iPhone SE, iPhone 14）で全ページが正常表示
- [ ] タブレット（iPad, iPad Pro）で全ページが正常表示
- [ ] 目次がモバイルでもアクセス可能
- [ ] コードブロックが横スクロールで閲覧可能
- [ ] タップ可能要素が44px以上

## 実装対象ファイル

1. `src/components/content/TableOfContents.tsx` - モバイル対応
2. `src/components/content/CodeBlock.tsx` - タッチ最適化
3. `src/components/layout/Header.tsx` - モバイル検索
4. `src/pages/Section.tsx` - レイアウト調整
5. `src/styles/globals.css` - 共通スタイル追加
