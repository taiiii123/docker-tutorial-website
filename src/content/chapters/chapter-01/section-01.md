# Dockerとは何か

## 概要

このセクションでは、Dockerの基本概念とコンテナ技術が解決する問題について学びます。

## Dockerとは

**Docker**（ドッカー）は、アプリケーションを**コンテナ**と呼ばれる軽量で独立した実行環境にパッケージ化し、どこでも同じように動作させることができるプラットフォームです。

2013年にDocker社（旧dotCloud社）によって開発され、現在ではソフトウェア開発における標準的なツールとなっています。

## なぜDockerが必要なのか

### 従来の問題：「私の環境では動くのに...」

開発者なら一度は経験したことがある問題：

- 開発環境では正常に動作するのに、本番環境ではエラーが発生する
- チームメンバーごとに異なる環境設定で、同じコードが異なる動作をする
- 新しい開発者がプロジェクトに参加する際、環境構築に何日もかかる

### Dockerによる解決

Dockerを使えば、アプリケーションとその実行に必要なすべての依存関係（ライブラリ、設定ファイル、システムツールなど）を1つのパッケージにまとめられます。

```dockerfile
# 例：Node.jsアプリケーションのDockerfile
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

このDockerfileがあれば、誰でも同じ環境でアプリケーションを実行できます。

## コンテナ技術の基本概念

### コンテナとは

**コンテナ**は、アプリケーションとその依存関係を隔離された環境で実行するための技術です。

主な特徴：

| 特徴 | 説明 |
|------|------|
| 軽量 | ホストOSのカーネルを共有するため、仮想マシンより軽量 |
| 高速 | 起動が秒単位で完了する |
| ポータブル | どの環境でも同じように動作する |
| 隔離 | 他のコンテナやホストシステムから独立 |

### イメージとコンテナの関係

- **イメージ（Image）**: アプリケーションの実行に必要なすべてを含むテンプレート（読み取り専用）
- **コンテナ（Container）**: イメージから作成された実行中のインスタンス

```
┌─────────────────┐
│     イメージ     │ ← 設計図（テンプレート）
└────────┬────────┘
         │ docker run
         ▼
┌─────────────────┐
│    コンテナ      │ ← 実行中のインスタンス
│  (実行環境)      │
└─────────────────┘
```

## Dockerの主なユースケース

### 1. 開発環境の統一

チーム全員が同じ開発環境を使用できます。

```bash
# docker-compose.ymlで開発環境を定義
docker compose up -d
```

### 2. マイクロサービスアーキテクチャ

各サービスを独立したコンテナとして実行できます。

```
┌─────────┐   ┌─────────┐   ┌─────────┐
│   Web   │   │   API   │   │   DB    │
│Container│ → │Container│ → │Container│
└─────────┘   └─────────┘   └─────────┘
```

### 3. CI/CDパイプライン

一貫したビルド・テスト環境を提供します。

### 4. 本番環境へのデプロイ

開発環境と同じ設定で本番環境にデプロイできます。

## Dockerのエコシステム

| コンポーネント | 説明 |
|---------------|------|
| Docker Engine | コンテナの実行エンジン |
| Docker Hub | 公開イメージのレジストリ |
| Docker Compose | 複数コンテナの管理ツール |
| Docker Desktop | Windows/Mac向けGUIツール |
| Kubernetes | コンテナオーケストレーション |

## まとめ

- Dockerはアプリケーションをコンテナ化するプラットフォーム
- コンテナは軽量・高速・ポータブルな実行環境
- 「どこでも同じように動く」環境を実現
- 開発からデプロイまで一貫した環境を提供
